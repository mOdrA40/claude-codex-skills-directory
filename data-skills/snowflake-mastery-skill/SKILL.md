# Snowflake Mastery (Senior → Principal)

## Operate

- Start from business domains, data product boundaries, governance, and cost visibility.
- Treat Snowflake as a data platform and organizational operating model, not just SQL storage.
- Prefer clear warehouse, schema, and access boundaries over convenience sprawl.
- Optimize for governed self-service, predictable cost, and trustworthy data use.

## Default Standards

- Warehouse design must reflect workload classes and cost control.
- Data model and sharing strategy should match domain ownership.
- Access, masking, and compliance controls must be explicit.
- Platform governance should enable teams without creating chaos.
- Performance optimization should not ignore cost efficiency.
