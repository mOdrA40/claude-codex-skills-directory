# Multi-Team Automation Governance

## Rules

- Shared automation ecosystems need ownership, standards, and deprecation policy.
- Platform teams should define role quality bars, secret handling norms, and execution safety expectations.
- Team autonomy should not create fleet-wide inconsistency or risk.
- Governance should improve clarity and scale, not only add ceremony.

## Governance Guidance

- Track owners of critical roles and inventories.
- Distinguish platform-certified automation from local team experiments.
- Review exception paths and privileged automations more strictly.
- Retire stale playbooks and ownerless roles deliberately.

## Principal Review Lens

- Which shared automation artifact has the highest blast radius?
- Are teams bypassing standards because the golden path is weak?
- What ownerless automation is most dangerous today?
- Is governance preventing incidents or merely documenting them?
